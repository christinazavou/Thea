int64 solve ( IMatrix< float64 > const *  m,
int8  compute_eigenvectors = true,
int64  num_requested_eigenpairs = -1,
IOptions const *  options = nullptr 
)
virtual

Find the eigenvalues (and optionally eigenvectors) of a matrix, optionally requesting a particular number of eigenpairs.

A set of additional options may be specified to control the solution process.

Parameters
mThe operator matrix for the eigenproblem. Must be square.
compute_eigenvectorsIf false, only the eigenvalues will be computed, not the eigenvectors. getEigenvalue() will fail on subsequent calls.
num_requested_eigenpairsThe number of eigenpairs to look for, which need not be the number returned. A negative value implies that there is no upper bound on the number of eigenpairs.
optionsBackend-specific options. See the documentation of each derived class wrapping a particular backend.
Returns
The number of eigenpairs found, or a negative number on error.

Valid options for the ARPACK backend, where n is the size of the operator matrix, are:

  • which: Part of eigenvalue spectrum to be returned
    • Type: string in {"LM", "SM", "LR", "SR", "LI", "SI"}
    • Default: "LM"
  • ncv: Number of Arnoldi vectors generated at each iteration (must be between num_requested_eigenpairs + 1 and n - 1)
    • Type: integer
    • Default: min{2 * num_requested_eigenpairs + 1, n - 1}
  • maxit: Maximum number of Arnoldi update iterations allowed
    • Type: integer
    • Default: 100 * num_requested_eigenpairs
  • tol: Stopping criterion, satisfied for a computed eigenvalue x if |x - x'| <= tol * |x|, where x' is the actual eigenvalue closest to x
    • Type: floating-point
    • Default: machine precision
  • sigma: See shift-invert setting
    • Type: floating-point
    • Default: 0.0
